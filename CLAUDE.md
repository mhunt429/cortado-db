## What This Project Is

`cortado-db` is a Scala 3 library that provides two things:
1. **Flyway SQL migrations** — versioned schema files that define the Cortado PostgreSQL database
2. **A Slick-generated data model** (`CortadoSchema.scala`) — auto-generated Scala table/row classes consumed by other services

The library is published to GitHub Packages and consumed by other services in the Cortado ecosystem.

## Commands

```bash
sbt test          # Run Flyway migration tests (requires PostgreSQL on localhost:65432)
sbt run           # Regenerate CortadoSchema.scala from a running DB (requires PostgreSQL on localhost:55432)
sbt publish       # Manually publish to GitHub Packages (requires GITHUB_TOKEN env var)
sbt "release with-defaults"  # Full sbt-release: bump version, tag, publish, push
```

## Database Connections

| Context | Host | Port | Database | User |
|---------|------|------|----------|------|
| Tests (`src/test/resources/application.conf`) | localhost | 65432 | postgres | postgres |
| Schema codegen (`TableGenerator.scala`) | localhost | 55432 | postgres | postgres |

PostgreSQL password is `postgres` in both cases.

## Architecture

### Migration files
SQL migrations live in `src/main/resources/db/migration/` and follow Flyway versioned naming (`V1__`, `V2__`, etc.). Flyway is configured with `outOfOrder(true)` and `cleanDisabled(false)`.

### Schema codegen workflow
`sbt run` triggers `Main.scala` → `MigrationRunner.runMigrations()` then `TableGenerator.generate()`. `TableGenerator` connects directly via JDBC (not `application.conf`) and uses Slick's `SourceCodeGenerator` with a custom subclass (`CustomSlickGenerator`) that maps `java.sql.Timestamp` columns to `java.time.Instant`.

**Do not hand-edit `CortadoSchema.scala`** — it is fully auto-generated by `sbt run`.

### Release process
GitHub Actions (`.github/workflows/publish.yml`) runs `sbt "release with-defaults"` on every push to `main`, which uses `sbt-release` to strip `-SNAPSHOT`, commit, tag, publish, bump to next snapshot, and commit again. The CI spins up a Postgres service container on port 65432 to satisfy the tests.

### Version management
The current version is tracked in `version.sbt` (managed by `sbt-release`). Do not edit `version.sbt` manually.
